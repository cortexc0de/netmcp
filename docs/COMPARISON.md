# Сравнительный анализ MCP-серверов для сетевого анализа

## Обзор

Экосистема Model Context Protocol (MCP) активно развивается, и в области сетевого анализа
уже появилось несколько реализаций MCP-серверов, интегрирующих такие инструменты, как
Wireshark/tshark, nmap и различные базы данных угроз. Данный документ представляет собой
честное и детальное сравнение четырёх основных проектов в этой нише.

Цель анализа — помочь пользователям и разработчикам выбрать инструмент, наиболее
подходящий для их задач, с учётом безопасности, функциональности, качества кода и
зрелости проекта.

---

## Сравниваемые проекты

### 1. netmcp (Python)

Полнофункциональный MCP-сервер для сетевого анализа с акцентом на безопасность
и production-ready качество.

- **25 инструментов**, 3 ресурса, 5 промптов
- 89% покрытие тестами, ~270 тестов
- 5-уровневая модель безопасности
- CI/CD pipeline на базе GitHub Actions
- GeoIP-обогащение через MaxMind
- Анализ HTTP-заголовков (токены, cookies, security headers)
- Аудит-логирование всех операций
- Rate limiting со скользящим окном
- `shell=False` во всех вызовах subprocess (защита от инъекций)
- Модульная архитектура: `core` → `interfaces` → `tools`
- Транспорт: stdio, SSE, streamable-http
- Docker-поддержка

### 2. WireMCP (Node.js/TypeScript)

Легковесный MCP-сервер на Node.js с минималистичным подходом.

- ~7 инструментов
- 0 тестов
- Критическая уязвимость command injection (`shell=true`, отсутствие валидации ввода)
- Извлечение Kerberos-хешей
- Интеграция с URLhaus
- Простая установка через `npx`
- Монолитная архитектура

### 3. Wireshark-MCP (Python)

Зрелый MCP-сервер с хорошей архитектурой и обширной документацией.

- ~19 инструментов
- Частичное покрытие тестами
- Хорошая модель безопасности
- 8 файлов документации
- Нет CI/CD
- Модульная архитектура
- Интеграция с nmap

### 4. wireshark-mcp (Python/PyShark)

Альтернативная реализация, использующая PyShark вместо прямых вызовов tshark.

- ~17 инструментов
- 1 тест
- Минимальная безопасность
- Основан на PyShark (не subprocess tshark)
- DNS-анализ
- Expert Info
- Анализ целевого трафика

## Анализ по категориям

### Безопасность

Безопасность — ключевой аспект для любого инструмента, работающего с сетевым трафиком
и выполняющего системные команды. Между проектами существует значительный разрыв
в подходах к обеспечению безопасности.

**netmcp** реализует 5-уровневую модель безопасности:

1. **Валидация входных данных** — все параметры проверяются до выполнения любых операций.
2. **Безопасное выполнение команд** — `shell=False` используется во всех вызовах
   `subprocess`, что исключает возможность инъекций через shell-интерпретатор.
3. **Rate limiting** со скользящим окном — предотвращает злоупотребление API.
4. **Аудит-логирование** — каждая операция фиксируется для последующего анализа.
5. **Контроль доступа к файлам** — ограничение путей и операций с файловой системой.

Это наиболее зрелая модель безопасности среди всех рассматриваемых проектов.

**WireMCP** имеет **критическую уязвимость**: использование `shell=true` при выполнении
команд tshark без надлежащей валидации входных данных. Это создаёт вектор атаки через
command injection — злоумышленник может сконструировать параметры, которые будут
исполнены как произвольные команды в shell. Отсутствие тестов усугубляет ситуацию,
поскольку уязвимость не может быть обнаружена автоматически.

**Wireshark-MCP** демонстрирует хорошую модель безопасности с корректным использованием
`shell=False` и валидацией входных данных. Однако отсутствуют rate limiting и
аудит-логирование, что ограничивает пригодность для production-сред.

**wireshark-mcp** обеспечивает лишь минимальную безопасность. Использование PyShark
частично снижает риски, связанные с прямым вызовом subprocess, однако валидация
входных данных реализована не в полном объёме.

### Тестирование и качество кода

Качество тестирования напрямую влияет на надёжность инструмента и уверенность
в корректности его работы.

**netmcp** лидирует с ~270 тестами и 89% покрытием кода. Тестовый набор включает
unit-тесты, интеграционные тесты и тесты безопасности. Наличие CI/CD pipeline
на базе GitHub Actions обеспечивает автоматическую проверку при каждом изменении.
Это единственный проект в сравнении, имеющий полноценный CI/CD.

**WireMCP** не содержит ни одного теста. Для инструмента, работающего с системными
командами и сетевым трафиком, это представляет серьёзный риск. Любые изменения
в коде не проходят автоматическую верификацию.

**Wireshark-MCP** имеет частичное покрытие тестами. Тесты присутствуют, но их
количество и охват недостаточны для полной уверенности в стабильности.
CI/CD отсутствует.

**wireshark-mcp** содержит всего 1 тест, что фактически равнозначно отсутствию
тестирования. Сложно говорить о надёжности при таком уровне покрытия.

### Функциональность

По количеству инструментов **netmcp** занимает первое место (25), предлагая наиболее
широкий набор возможностей:

- **GeoIP-обогащение** через MaxMind — уникальная функция, позволяющая определять
  географическое расположение IP-адресов прямо в контексте анализа.
- **Анализ HTTP-заголовков** — извлечение и анализ токенов аутентификации, cookies,
  security headers. Ни один другой проект не предлагает подобной функциональности.
- **5 промптов** — готовые шаблоны для типовых сценариев анализа, снижающие порог входа.
- **6 инструментов nmap** — полноценная интеграция с nmap для сетевого сканирования.
- **Threat Intelligence** — интеграция с URLhaus и AbuseIPDB для проверки индикаторов
  компрометации.

**WireMCP** предлагает минималистичный набор (~7 инструментов), но включает полезную
функцию извлечения Kerberos-хешей в формате, совместимом с hashcat. Интеграция
с URLhaus обеспечивает базовую проверку угроз.

**Wireshark-MCP** занимает второе место (~19 инструментов) и предлагает сбалансированный
набор возможностей, включая интеграцию с nmap. Наличие 3 ресурсов и 3 промптов
делает его достаточно полным MCP-сервером.

**wireshark-mcp** обладает ~17 инструментами и выделяется двумя уникальными
возможностями, отсутствующими в других проектах:

- **DNS-анализ** — специализированные инструменты для работы с DNS-трафиком.
- **Expert Info** — доступ к механизму Expert Info из Wireshark, предоставляющему
  диагностическую информацию о проблемах в трафике.

Эти функции делают wireshark-mcp ценным инструментом для определённых сценариев
анализа.

### Архитектура

Архитектурные решения определяют масштабируемость и сопровождаемость проекта.

**netmcp** реализует чистую модульную архитектуру с чётким разделением на слои:
`core` (базовая логика) → `interfaces` (абстракции взаимодействия) → `tools`
(реализация инструментов). Такая структура облегчает добавление новых инструментов
и тестирование отдельных компонентов. Поддержка трёх транспортов (stdio, SSE,
streamable-http) обеспечивает гибкость развёртывания.

**Wireshark-MCP** также следует модульному подходу, что положительно сказывается
на качестве кода и возможности расширения.

**WireMCP** имеет монолитную архитектуру. Для проекта с ~7 инструментами это
допустимо, однако масштабирование в текущей структуре будет затруднено.

**wireshark-mcp** выделяется использованием PyShark вместо прямого вызова tshark
через subprocess. Это принципиально иной подход: PyShark предоставляет
объектно-ориентированный доступ к данным пакетов, что может быть удобнее для
определённых задач, но добавляет дополнительную зависимость и может ограничивать
контроль над выполнением tshark.

### Документация

Качественная документация критична для MCP-серверов, так как пользователи должны
понимать доступные инструменты и их параметры.

**Wireshark-MCP** формально лидирует по количеству файлов документации (8), покрывая
различные аспекты проекта.

**netmcp** предлагает комплексную документацию: README, API reference, описание
архитектуры, а также данный сравнительный документ. Документация ориентирована
как на пользователей, так и на разработчиков.

**WireMCP** и **wireshark-mcp** ограничиваются README-файлами, что может быть
недостаточно для полноценного освоения инструментов.

## Честная критика netmcp

Несмотря на лидерство по многим критериям, netmcp имеет ряд объективных недостатков,
которые необходимо учитывать:

- **Нет DNS-анализа** — специализированные инструменты для работы с DNS-трафиком
  отсутствуют, тогда как wireshark-mcp предлагает полноценное решение в этой области.
- **Нет Expert Info анализа** — механизм Expert Info из Wireshark, предоставляющий
  ценную диагностическую информацию, реализован только в wireshark-mcp.
- **Нет прямой интеграции с PyShark** — проект использует subprocess для вызова
  tshark, что является более низкоуровневым подходом по сравнению с
  объектно-ориентированным API PyShark.
- **Проект моложе конкурентов** — несмотря на зрелую кодовую базу, netmcp появился
  позже других проектов, что означает меньший объём реального использования
  в production-средах.
- **Большее количество зависимостей** — расширенная функциональность (GeoIP, rate
  limiting, аудит) влечёт за собой большее число зависимостей, что может усложнять
  развёртывание в ограниченных средах.

## Заключение

Каждый из рассмотренных проектов занимает свою нишу в экосистеме MCP-серверов
для сетевого анализа.

**netmcp** является наиболее полным и production-ready решением. Он лидирует по
безопасности (5-уровневая модель), тестированию (89% покрытие, ~270 тестов),
количеству инструментов (25) и инфраструктуре (CI/CD, Docker). Для команд,
которым требуется надёжный и безопасный инструмент для повседневного использования,
это оптимальный выбор.

**Wireshark-MCP** — сильный второй кандидат с хорошей архитектурой, достойной
безопасностью и обширной документацией. Подходит для пользователей, предпочитающих
зрелый проект с проверенной кодовой базой.

**wireshark-mcp** заслуживает внимания благодаря уникальным возможностям DNS-анализа
и Expert Info, а также альтернативному подходу через PyShark. Для задач, связанных
с глубоким анализом DNS-трафика, он может оказаться наиболее подходящим инструментом.

**WireMCP** в текущем состоянии **не рекомендуется для production-использования**
из-за критических уязвимостей безопасности и полного отсутствия тестов. Однако
простота установки через `npx` и наличие Kerberos-функциональности делают его
пригодным для быстрого прототипирования в изолированных средах.

В конечном счёте, выбор инструмента зависит от конкретных требований: приоритет
безопасности и полноты → netmcp, DNS-анализ → wireshark-mcp, зрелость и документация →
Wireshark-MCP.
